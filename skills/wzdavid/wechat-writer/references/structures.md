# 微信公众号写作参考：结构模板与标题（大卫风格）

## 文章结构模板

根据选定的文章类型选择对应结构：

### A. 知识分享型（介绍方法论/框架）

```
# 标题（数字型或疑问型）

[开篇故事，100-200字，第一人称，有时间地点人物]

[点出核心问题/价值，1段，承接故事引出主题]

## 一、[框架名称/第1个要点]
[概念解释，100字内]
[案例或比喻，100-150字]

## 二、[第2个要点]
...（同上）

## 三、[第3个要点]
...（同上）

## 写在最后
[实践建议，1段]
[金句或名言收尾，30-50字]
```

### B. 案例分析型（分析事件/项目）

```
# 标题（"从XX事件看YY"或"XX带给我们的启示"）

[事件引入，背景+关键事实，150-200字]

## 这件事真正的问题在哪里
[从实践者角度切入，1-2个角度]

## [分析角度1]
[结合自身经验，150-200字]

## [分析角度2]
...

## 给我们的启示
[实操性建议，分点或分段]

[结尾点睛，回扣开篇]
```

### C. 观点输出型（对趋势/现象的看法）

```
# 标题（观点型，可引发共鸣或轻度争议）

[开篇现象/痛点，引共鸣，100字]
[亮出核心观点，1句话，加粗]

## [论点1]
[观点句 + 数据/案例 + 个人感悟，200字]

## [论点2]
...

## [论点3]
...

## 我的判断
[对未来的预见或行动建议]

[收尾：简短有力，1-2句话]
```

### D. 技术实操型（工具介绍/动手教程）

**适用场景**：介绍某个技术工具、软件配置、实操步骤，类似"在笔记本上实现本地RAG""轻松使用Gemma模型"这类文章。

**风格特征**：
- 结构以**数字编号步骤**为主干（1. 下载 → 2. 配置 → 3. 使用）
- 每节**短小精悍**，100-200字，直接说做什么，不铺垫
- **工具名、路径、代码**具体准确（存储路径、Docker命令等）
- **代码块规范**：命令直接给出，不加废话解释
- **第一人称实操感**："我创建了一个工作区""我就直接选用nomic-embed-text"
- **轻评论**：偶尔一句个人判断，不展开（"这一点对企业级应用特别关键""值得称许的是……"）
- **开篇承接上文**（如有系列）：简短说明这篇文章解决什么问题，1-2句话即可

```
# 标题（"在XX上实现YY"/"轻松使用XX"/"XX入门"）

[背景/问题，1-3句话，说清楚这篇文章解决什么]

[图片：产品/工具截图]

## 1. [第一步操作名称]
[做什么，核心说明，50-100字]
[如有代码/命令：直接给出代码块]
[个人补充（可选）："我就直接选用……""以我的情况为例……"]

## 2. [第二步操作名称]
...（同上）

## 3. [第三步操作名称]
...（同上）

[结尾（可选）：1-2句话点评效果或展望，如"未来还可以在XX方面进一步调优"]
```

**注意**：
- 技术类文章**不需要**长故事开篇，直接进入主题
- 每步之间逻辑连贯（先装工具→再配置→再使用）
- 截图/配图需求较多，在需要图的地方标注 `[图片：描述]`
- 个人判断点到即止，不深入展开分析

### E. 深度技术分析型（大模型/AI技术深度解析）

**适用场景**：对某个技术领域进行系统性分析，如"ChatGLM金融大模型解析""大模型应用架构设计"，既有技术深度又有方法论提炼。

**风格特征**：
- 开篇：背景说明 + 引出核心问题（2-3段，200字内）
- 结构：**编号段落**（ 1  2  3 而非1. 2. 3.），每段一个核心主题
- **框架图先行**："按照惯例，我研究分析……画了一张图"——先给总体框架图
- 逐点拆解：每个编号段落深入讲一个技术点，有代码/表格/示例
- 结尾："X点启示"或"后续开发计划"，总结提炼

```
# 标题（"XX的三点启示"/"XX解析"/"一图读懂XX"）

[背景说明，1-2段，引出要解决的核心问题]

[图片：总体框架图]

按照惯例，我研究分析……画了一张图。这个框架包含X大块内容……

 1 
核心流程/架构

[详细讲解，200-400字]
[图片/表格/代码示例]

 2 
关键技术点1

[详细讲解，200-400字]

 3 
关键技术点2

[详细讲解，200-400字]

...

 N 
X点启示 / 后续计划

[总结提炼，3-5条]
```

**注意**：
- 编号用空格包围的数字（ 1  2  3 ），这是大卫深度技术文的标志性格式
- 必须有框架图（"按照惯例，我画了一张图"）
- 结尾必须提炼方法论（"X点启示""三个关键能力"）

### F. 故事记叙型（亲历事件完整记录）

**适用场景**：记录某个有意义的亲身经历，如"七二一大暴雨""某次项目经历"，强调故事性和细节还原。

**风格特征**：
- 开篇直接进入时间地点（"那天是2012年7月21日……"）
- 细节丰富：对话、心理活动、环境描写
- 第一人称叙事，有"我"的视角和感受
- 时间线清晰（"晚上八点半""后来才知道"）
- 结尾升华：从个人经历到普遍思考

```
# 标题（事件主题，可含时间/地点）

[开篇：直接进入场景，时间+地点+事件]
那天是XXXX年X月X日……

[故事发展：按时间线推进]
## [可选小标题，或不分节]
[细节描写，有对话、心理、环境]
"……"（直接引语对话）
我心想/我注意到/我才发现……（心理活动）

[故事高潮]
[关键场景细节]

[结尾：回到现在 + 反思升华]
现在看来……
然而……这些代价难道是真的不可避免的吗？
```

**注意**：
- 大量细节：具体的时间、对话、动作、环境
- 有温度：个人情感和判断自然流露
- 结尾必须升华：从"我的故事"到"对所有人的启发"

### G. 年度总结型（个人年度回顾）

**适用场景**：年度总结、阶段性回顾，如"我的2023年总结"。

**风格特征**：
- 开篇：简短概括这一年的关键词（1-2段）
- 结构：数字编号分板块（1/ 工作 2/ 读书 3/ 写作 4/ 徒步）
- 每板块：数据 + 具体事件 + 个人感受
- 真实感：有具体地点、人名、书名、数字
- 结尾：展望下一年，简短祝福

```
# 我的XXXX年总结

[图片：年度代表性图片]

[开篇总结，1-2段，概括这一年的关键词和整体感受]

1/ 板块1（如：工作）

[这一年在这方面的主要事件]
[具体数据：去了X个城市、见了X个人、做了X个项目]
[个人感受和收获]

## 小标题1（可选，如：设计数字化转型框架）
[具体成果]
[图片：相关图片]

## 小标题2
...

2/ 板块2（如：读书）

[数据：今年读了X本书]
[图片：书籍封面]
[书单列表]
[推荐的书 + 简短评价]

3/ 板块3
...

[结尾：展望]
XXXX年即将到来，祝朋友们……
```

**注意**：
- 必须有真实数据（多少本书、多少篇文章、去过哪些地方）
- 语气平实、不夸张，适度谦逊
- 可以有一些小遗憾（"今年徒步活动比较少""没能在7小时内完成"）
- 结尾简短祝福即可，不抒情

## 标题写法参考

| 类型 | 示例 |
|------|------|
| 数字型 | 流程优化的五个方法 / 流程文档设计的14个要素 / 金融大模型的三点启示 |
| 疑问型 | 你有流程思维吗？/ AI真的能替代管理者吗？|
| 叙述型 | 从腾讯老干妈事件看内部控制 / 从实践者视角看IBM流程体系 |
| 故事型 | 一场愉快的流程优化轻咨询 / 我在IBM第一天学到的事 |
| 实操型 | 在笔记本电脑上，轻松使用Gemma模型 / 实现本地知识库和大模型RAG |
| 分析型 | 一图读懂金融大模型 / ChatGLM金融大模型挑战赛解析 |
| 记叙型 | 担心的事终会发生－记忆中的"七二一"大暴雨 |
| 总结型 | 我的2023年总结 |
