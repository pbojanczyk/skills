# 微信公众号写作参考：质检、爆款优化与领域要点（大卫风格）

## 写作初稿（通用提醒）

- 优先使用用户提供的真实素材和个人经历
- 联网搜索到的信息用于补充数据和背景，不替代个人观点
- 初稿不追求完美，重点保证逻辑通顺和风格到位
- 在需要配图的位置标注 `[图片：说明]`（建议全文配 3-6 张）

## 三遍审校（必做）

### 第一遍：基础检查
- [ ] 错别字、标点符号（中文标点，不用英文标点）
- [ ] 数字格式统一（汉字数字 vs 阿拉伯数字）
- [ ] 段落格式（段与段之间空行，标题层级正确）
- [ ] 字数是否在 1500-3000 字区间

### 第二遍：逻辑与表达
- [ ] 开篇故事是否有时间/人物/细节？
- [ ] 每个观点是否有对应案例支撑？
- [ ] 有无"AI腔"套话需要替换？
- [ ] 自问自答结构是否自然？
- [ ] 中英术语是否标注了中文解释？
- [ ] 是否存在“无来源的大词”（研究表明/据统计/专家指出）需要补来源或删改？

### 第三遍：风格与完读率
- [ ] **前100字**：是否在3秒内抓住读者？有无悬念或价值承诺？
- [ ] **中段**：每300字是否有新洞察或小转折？
- [ ] **结尾**：是否有力收束？是否回扣开篇？
- [ ] 整体语气是否一致（实践者/平等/温度）？
- [ ] 是否避免了说教和鸡汤？

## 爆款结构优化（可选）

| 公式 | 适用类型 | 结构 |
|------|----------|------|
| 问题→原因→方案→升华 | 知识分享型 | 发现问题 → 分析根因 → 给出方法 → 更高视角总结 |
| 故事→冲突→转折→洞察 | 案例分析型 | 引入故事 → 出现矛盾 → 意外转折 → 提炼洞见 |
| 颠覆认知→新视角→重建 | 观点输出型 | 打破常识 → 提供新框架 → 帮读者重建认知 |
| 数据冲击→深挖→案例→升华 | AI/行业分析 | 用数据震惊读者 → 解释为什么 → 案例实证 → 个人判断 |
| 问题→步骤→效果→延伸 | 技术实操型 | 说清楚解决什么问题 → 分步操作 → 展示结果 → 展望下一步 |
| 框架→拆解→案例→启示 | 深度技术分析型 | 画框架图 → 编号逐点讲解 → 代码/表格示例 → 提炼X点启示 |
| 场景→细节→高潮→反思 | 故事记叙型 | 进入场景 → 丰富细节描写 → 关键转折 → 升华到普遍思考 |
| 概括→分板块→数据→展望 | 年度总结型 | 整体关键词 → 按领域分块 → 具体数据和事件 → 简短展望 |

**完读率关键检查**：
- **开头（前100字）**：3秒内抓住注意力，设置悬念或明确价值承诺
- **中段**：每300字有新洞察或转折，不能平铺直叙
- **结尾**：主题升华，有行动引导或抛出思考问题给读者

## 领域写作要点

**AI/科技类**：联网搜索最新进展 → 将技术转化为"对企业管理/职场人的影响和启示" → 避免纯技术介绍，要有人的视角

**技术实操类**：步骤清晰、工具名准确、代码可直接用 → 不需要长故事开篇 → 每步加一句个人实操感受 → 结尾说明效果和局限，展望下一步

**深度技术分析类**：必须有框架图（"按照惯例，我画了一张图"）→ 编号段落（ 1  2  3 ）→ 结尾提炼方法论（"X点启示"）→ 专业术语多但都解释清楚

**管理/流程类**：引用IBM经验、流程咨询案例 → 侧重"实战中遇到的问题"而非教科书式解释

**职场/个人成长类**：用"我的困惑→我如何想通的"叙事结构 → 展示决策过程（犹豫→思考→决定）→ 数据后加个人判断

**故事记叙类**：开篇直接进场景（时间+地点）→ 大量细节（对话、心理、环境）→ 结尾必须从个人经历升华到普遍思考

**年度总结类**：数字编号分板块（1/ 2/ 3/）→ 必须有真实数据 → 语气平实、可有小遗憾 → 结尾简短祝福

## 快速参考：常见请求处理

| 用户请求 | 执行路径 |
|----------|----------|
| "帮我写一篇关于XX的文章" | 确认需求 → 必要时搜索 → 选题/大纲 → 写作 → 三遍审校 |
| "帮我写一篇XX工具的教程/介绍" | 搜索（必做）→ 选技术实操型 → 写作（步骤结构）→ 三遍审校 |
| "我不确定写什么，给我选题" | 搜索热点 → 给选题卡片 → 定大纲/写作 |
| "帮我优化/审校这篇文章" | 三遍审校 → 产出修改稿 → 可选爆款优化 |
| "帮我改得更有我的风格" | 读取风格规范 → 逐段改写 → 三遍审校 |
