# 微信公众号写作 Skill（大卫风格）

## 概述

本 Skill 为大卫的公众号「大卫数智话」提供全流程创作辅助，涵盖需求确认、信息搜索、选题讨论、初稿写作、三遍审校、爆款优化全流程。文章主要领域：**科技/AI/企业管理/职场成长**。

**流程顺序强制规定**：遇到新概念/新产品/新技术 → 必须先完成搜索 → 再进行选题 → 再写作。严禁跳步。

## 使用模式（自动化写作友好）

### 模式 A：交互式（默认）

先确认关键约束 → 必要时联网检索 → 给 3-4 个选题卡片 → 用户选定后出稿 → 三遍审校 → 输出最终稿。

### 模式 B：一键直出（用户明确要求“直接写/直接出稿/不要问我”）

不再追问，仅做最小假设并在正文前用一段话列出假设（不超过 80 字），然后直接出完整稿；如涉及新概念/新事件仍需先联网检索再出稿。

### 支持的常见意图

- 写新文章（从 0 到 1 出稿）
- 选题/定大纲（只给方向与大纲，不写全文）
- 改稿/润色（用户提供原文，输出修改后版本）
- 审校（对原文做三遍审校并给修改稿）
- 标题/开头/结尾优化（局部增强完读率）

### 输入模板（推荐）

写新文章：

```
主题：
触发场景（可选）：
目标读者（可选）：
文章类型（可选）：
字数（可选）：
我可提供的素材（可选）：
是否需要联网检索（默认：需要，除非明确“不要联网”）：
是否需要配图位（默认：需要）：
```

改稿/审校：

```
目标：更有大卫风格 / 更通顺 / 更有观点 / 更像爆款 / 更短 / 更长（可多选）
改动强度：轻 / 中 / 重（默认：中）
必须保留的信息：
必须删除的内容：
原文：
```

### 兜底规则（信息不全时）

- 未给文章类型：默认知识分享型
- 未给字数：默认 1800-2400 字
- 未给读者：默认“企业管理者 + 职场人”
- 未给素材：用一个合理的第一人称场景开篇，但不捏造可核查事实

## Step 0：加载写作约束（⭐ 强制执行）

每次运行都必须先加载写作约束，再进入后续步骤。写作约束来自两部分：references（规则）+ assets（样例）。

### 0.1 references：本次写作必须读取哪些规则

- 写新文章：必须读取 `references/style.md` 与 `references/structures.md`
- 选题/定大纲：必须读取 `references/structures.md`
- 改稿/润色：必须读取 `references/style.md` 与 `references/quality.md`
- 审校：必须读取 `references/quality.md`
- 标题/开头/结尾优化：必须读取 `references/style.md`

### 0.2 assets：本次写作必须对齐哪些过往样例

每次运行必须从 `assets/` 中选择 2-3 篇最相近的过往文章做风格对齐：

- 优先同主题/同类型：技术实操对技术实操，观点输出对观点输出
- 优先同结构密度：长文/短文、分节多/分节少
- 如果不确定，就选 1 篇技术文 + 1 篇管理文 + 1 篇故事文作兜底

加载完成后，必须先输出一段“本次写作约束摘要”（不超过 120 字），包含：标题句式、开头方式、段落节奏、结尾收束方式。

---

## Step 1：确认需求

收到写作请求，先向用户确认（如信息已足够可跳过）：

- **主题/方向**：这篇文章想聊什么？
- **触发场景**：是最近遇到的事、看到的新闻、还是想分享某个经验？
- **目标读者**：主要写给谁看？（企业管理者/职场人/科技爱好者）
- **字数要求**：通常 1500-3000 字，是否有特殊要求？
- **有无素材**：是否有可引用的个人经历、具体案例或数据？

---

## Step 2：信息搜索（⭐ 新概念/新技术/新事件时强制执行）

**触发条件**：主题涉及近期 AI 产品、行业新动态、最新报告、近期事件等。

**操作步骤**：
1. 用 `WebSearch` 搜索 2-3 组关键词（保持简洁，1-4 个词）
2. 用 `WebFetch` 获取重要页面全文（官方文档、权威媒体报道优先）
3. 整理关键信息：
   - 核心功能/事实是什么
   - 有哪些值得引用的数据或案例
   - 行业/用户的主要反馈和争议点
4. 向用户简要汇报搜索结论，确认后进入选题

**搜索质量标准**：优先官方文档 > 权威媒体 > 行业报告 > 用户评价，避免引用论坛传言。

**引用与事实规则（自动化必备）**：
- 涉及“时间/数据/机构结论/产品能力/政策规则”等可核查事实，必须在文末给出来源链接清单（3-8 条）
- 无法从来源确认的内容，一律改写为“我的判断/可能/推测”，或明确标注“待确认”

---

## Step 3：选题讨论（⭐ 必做，不直接开写）

基于需求和搜索结果，提供 **3-4 个选题方向**，每个包含：

```
【选题 A】
标题：[吸引人，符合大卫标题风格]
核心角度：[一句话说清楚这篇文章的独特视角]
文章类型：知识分享型 / 案例分析型 / 观点输出型 / 技术实操型 / 深度技术分析型 / 故事记叙型 / 年度总结型
大纲：
  1. [开篇方向]
  2. [主体要点1]
  3. [主体要点2]
  4. [主体要点3]
  5. [结尾方向]
工作量：⭐⭐⭐（1-5星）
优势：[为什么适合大卫写]
挑战：[潜在难点]
```

等用户选定后再进入创作。

## 路由与工作流（按用户输入选择）

### 场景 1：给出主题与需求（可能需要检索）→ 写作完成

- 加载写作约束（references + assets）
- 确认需求（如信息足够可跳过）
- 若涉及新概念/新事件：先联网检索，再进入选题
- 给 3-4 个选题卡片（含标题、角度、类型、大纲）
- 按选定结构写完整初稿
- 三遍审校后输出最终稿

### 场景 2：给出主要内容/要点/原稿 → 风格化改写与完善

- 加载写作约束（references + assets）
- 读取用户提供原文，先明确目标与改动强度（轻/中/重）、必须保留/必须删除项
- 不新增可核查事实；若需要补数据或背景：先联网检索并在文末给来源
- 输出修改后全文；如改动较大，先给“修改要点清单”再给正文

### 场景 3：不确定写什么 → 检索热点 → 给选题

- 加载写作约束（references + assets）
- 联网检索最近热点（按领域拆 2-3 组关键词；优先权威媒体/官方发布）
- 先输出“热点摘要”（5-10 条，一句一条，附来源链接）
- 再输出“选题卡片”（6-10 个：标题、角度、适配读者、文章类型、可写大纲）
- 用户选定后回到场景 1 的写作流程

### 其他常见场景（同样支持）

- 只要大纲：执行到“选题卡片/大纲”即止
- 只要标题：给 10 个标题，并标 1 个推荐
- 局部优化：只改开头/结尾/标题/某一段（保持其余内容不变）
- 审校：执行三遍审校，输出“问题清单 + 修改后版本”

## 通用硬规则（自动化写作稳定性）

- 若主题包含可核查事实：按需联网检索，文末给 `## 参考来源`（3-8 条链接）
- 不捏造可核查事实；不确定则写成“我的判断/可能/推测/待确认”
- 风格优先级：真实经历与细节 > 清晰结构 > 案例支撑 > 轻评论 > 金句收束
- 禁用“AI腔套话/说教口吻/无来源大词/夸张标题/鸡汤结尾”

## 输出格式

- Markdown：`# 标题` + `##/###` 分节，段落间空一行
- 图片位置：`[图片：描述]`
- 若使用了联网信息：文末追加 `## 参考来源`

## 保存约定（自动化创作）

### 最终文章

- 保存目录：`articles/`
- 文件命名：`YYYY-MM-DD_<文章标题>.md`
- 同主题多版本：在文件名末尾追加 `_v2`、`_v3`

### 临时文件（一次创作一套）

- 保存目录：`.trae/wechat-writer/runs/<run_id>/`
- run_id 命名：`YYYYMMDD-HHMMSS_<主题关键词>`
- 文件清单（按需生成）：
  - `00_request.md`：用户输入与关键约束
  - `01_research.md`：检索结果摘要 + 参考来源链接
  - `02_topics.md`：选题卡片
  - `03_outline.md`：确定的大纲
  - `04_draft.md`：初稿
  - `05_edit.md`：改稿/润色版本（可多次）
  - `final.md`：最终稿备份（与 articles/ 中一致）

### 保存原则

- 默认落盘保存最终文章与临时文件；用户明确要求“不要保存/不落盘”时，只在对话中输出
- 临时文件可用于复盘与二次改稿，不自动删除；需要清理时按 run_id 整目录删除

## references 与 assets（强制使用）

每次运行必须执行 Step 0 的加载规则：

- `references/style.md`：风格规范 + 语感锚点
- `references/structures.md`：结构模板 + 标题类型
- `references/quality.md`：三遍审校 + 爆款优化 + 领域要点
- `assets/`：过往文章样例（每次选 2-3 篇做风格对齐）
