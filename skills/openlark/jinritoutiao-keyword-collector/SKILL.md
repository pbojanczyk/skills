# 今日头条关键词收集

## 功能概述

本技能通过浏览器自动化访问今日头条 (www.toutiao.com)，模拟用户在搜索框输入关键词的行为，自动收集搜索下拉提示框中显示的相关关键词建议。

## 使用场景

- SEO 关键词研究和优化
- 内容创作灵感收集
- 热点话题发现
- 竞品关键词分析
- 市场调研和趋势分析

## 操作流程

### 1. 启动浏览器

使用 `browser` 工具启动浏览器：

```
action: start
profile: openclaw
```

### 2. 访问今日头条首页

导航到今日头条：

```
action: navigate
targetUrl: https://www.toutiao.com
```

等待页面加载完成。

### 3. 获取页面快照

使用 `snapshot` 获取页面元素结构，识别搜索框位置：

```
action: snapshot
refs: aria
```

在快照结果中查找搜索框元素（通常包含"搜索"、"search"等关键词的 input 元素）。

### 4. 输入关键词

使用 `act` 工具在搜索框中输入关键词：

```
action: act
ref: <搜索框的 ref 值>
kind: type
text: <用户提供的关键词>
slowly: true
```

设置 `slowly: true` 以确保页面能够正确触发自动提示。

### 5. 等待提示框出现

输入完成后，等待 1-2 秒让自动提示框加载：

```
action: act
kind: wait
timeMs: 2000
```

### 6. 获取提示框快照

再次获取页面快照，这次包含自动提示框中的关键词列表：

```
action: snapshot
refs: aria
```

### 7. 提取关键词

从快照结果中提取提示框中的所有关键词文本。提示框通常包含多个 `option`、`li` 或 `div` 元素，每个元素包含一个建议关键词。

### 8. 输出格式
```
关键词：[输入的关键词]
搜索建议词：
1. 建议词1
2. 建议词2
3. 建议词3
...
```

### 9. 清理（可选）

如果用户没有进一步需求，可以关闭浏览器：

```
action: close
```

## 注意事项

1. **网络延迟**：今日头条页面加载可能需要时间，确保在操作前有足够等待时间
2. **元素定位**：使用 `refs: aria` 可以获得更稳定的元素引用
3. **输入速度**：设置 `slowly: true` 确保页面能正确触发自动提示
4. **反爬虫机制**：避免短时间内频繁请求，建议在多次收集之间间隔 5-10 秒
5. **会话保持**：如需多次收集，保持浏览器会话开启以提高效率

## 多关键词批量收集

如需收集多个关键词的建议，可以：

1. 保持浏览器会话开启
2. 对每个关键词重复步骤 4-7
3. 使用 `act` 的 `kind: type` 清空搜索框后输入新关键词
4. 将所有结果整理成一个汇总报告

## 故障排查

### 搜索框找不到

- 检查页面是否完全加载
- 尝试刷新页面后重试
- 使用更详细的快照（增加 `depth` 参数）

### 提示框不出现

- 确认输入速度足够慢（`slowly: true`）
- 增加等待时间（`timeMs: 3000` 或更长）
- 检查网络连接是否正常

### 关键词列表为空

- 确认输入的关键词有效
- 尝试更换常见关键词测试
- 检查今日头条是否有页面结构更新
